# mod10_T_FF 設計說明文件

本文件說明本專案如何使用 T 行正反器（T Flip-Flop）實作同步 mod10 計數器，並解釋程式邏輯與對應電路架構。

## 1. 專案檔案結構

- `T_FF.v`：單一 T 觸發器模組（正緣觸發、非同步低有效重置）。
- `mod10_T_FF.v`：由 4 個 T 觸發器組成的同步 mod10 計數器。
- `mod10_T_FF.qpf`、`mod10_T_FF.qsf`：Quartus 專案設定檔。
- `mod10_T_FF.v`（可能為 Quartus 自動產生或預設檔）：不作為本次核心實作主體。

## 2. 設計目標

- 使用 4 個 T 觸發器設計一個 4-bit 計數器。
- 計數序列限制為十進位 0 到 9，亦即：
  - 0000 (0) -> 0001 (1) -> ... -> 1001 (9) -> 0000 (0)
- 具備非同步低有效重置 `rst_n`：
  - `rst_n = 0` 時立即清零。
- 加入非法狀態（1010~1111）保護機制：
  - 若因雜訊或暫態進入 10~15，下一個時脈自動修正回 0。

## 3. T 觸發器工作原理（T_FF）

在 `T_FF.v` 中，狀態更新規則如下：

- 非同步重置優先：
  - `if (!rst_n) q <= 0`
- 正常時脈正緣觸發：
  - `t = 1`：`q` 反相（toggle）
  - `t = 0`：`q` 保持（hold）

其核心狀態方程可寫為：

- `Q_next = Q xor T`

此方程是後續推導 mod10 控制邏輯的關鍵。

## 4. mod10_T_FF 程式架構

`mod10_T_FF.v` 主要分為四個區塊：

1. 內部狀態位元：`q0, q1, q2, q3`
2. 狀態偵測：`is_9`、`invalid_state`
3. 控制訊號：`force_zero`
4. T 輸入邏輯：`t0, t1, t2, t3`

最後以四個 `T_FF` instance 實作 4-bit 同步計數器。

## 5. 狀態偵測與保護邏輯

### 5.1 偵測 9（1001）

- `is_9 = q3 & ~q2 & ~q1 & q0`

當狀態為 9，下一拍必須回到 0。

### 5.2 偵測非法狀態（10~15）

- 非法條件可簡化為：
  - `invalid_state = q3 & (q2 | q1)`

原因：

- 8、9 對應 1000、1001（`q3=1` 但 `q2=0` 且 `q1=0`）是合法。
- 10~15 必定滿足 `q3=1` 且 `q2=1` 或 `q1=1`。

### 5.3 強制清零條件

- `force_zero = is_9 | invalid_state`

當 `force_zero = 1` 時，下一拍應全部回到 0000。

## 6. T 輸入推導

### 6.1 正常二進位遞增（0~8）

同步遞增計數器的 T 輸入為：

- `t0 = 1`
- `t1 = q0`
- `t2 = q1 & q0`
- `t3 = q2 & q1 & q0`

### 6.2 強制回零（9 或非法狀態）

利用 `Q_next = Q xor T`，若希望下一拍 `Q_next = 0`，只要令：

- `T = Q`

則：

- `Q_next = Q xor Q = 0`

因此程式採用：

- `t0 = force_zero ? q0 : 1'b1`
- `t1 = force_zero ? q1 : q0`
- `t2 = force_zero ? q2 : (q1 & q0)`
- `t3 = force_zero ? q3 : (q2 & q1 & q0)`

這樣可同時滿足：

- 0~8 正常累加。
- 9 下一拍回 0。
- 10~15 下一拍自動修正回 0。

## 7. 電路方塊架構

可用下列概念圖理解（非 gate-level 細圖）：

```text
                 +-----------------------------+
clk -----------> |                             |
rst_n ---------> |      組合邏輯控制區         | ---> t0
q3 q2 q1 q0 ---->|  (is_9 / invalid / force)   | ---> t1
                 |                             | ---> t2
                 +-----------------------------+ ---> t3
                                |    |    |    |
                                v    v    v    v
                              +----+ +----+ +----+ +----+
clk -------------------------> |FF0| |FF1| |FF2| |FF3|
rst_n -----------------------> |TFF| |TFF| |TFF| |TFF|
                              +----+ +----+ +----+ +----+
                                |      |      |      |
                               q0     q1     q2     q3
                                \______ \______ \_____\
                                       4-bit q[3:0]
```

架構重點：

- 四顆 T_FF 共用同一個 `clk`，因此是同步計數器。
- `rst_n` 直接接到每顆 FF 的非同步重置端。
- 組合邏輯只決定下一拍各 FF 是否切換（T=1）或保持（T=0）。

## 8. 時序與行為摘要

- `rst_n=0`：立即輸出 `q=0000`。
- `rst_n=1` 且每個 `posedge clk`：
  - 若 `q` 在 0~8，下一拍加 1。
  - 若 `q=1001`（9），下一拍回 `0000`。
  - 若 `q` 在 10~15，下一拍也回 `0000`。

## 9. 設計優點

- 結構清楚：FF 與控制邏輯分離。
- 可綜合性高：全部為標準 Verilog 可綜合描述。
- 容錯性佳：可自動修正非法狀態。
- 教學價值高：清楚展示 T FF 在同步計數器中的應用方式。
